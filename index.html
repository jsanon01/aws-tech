Introduction to AWS

21st century is well-known for the digital age; needless to say that everything is based on information technology.

AWS has made History by running massive-scale technologies not only around the world with a technical presence in several geographical regions, availability zones, and edge-locations but also using "Products & Services" such as computing, networking, storage, and database.

  • I cannot talk about AWS without mentioning the following advantages of Cloud Computing:
    1. Gaining agility meaning to provision resources almost instantly;
    2. Avoiding guessing capacity meaning to scale up or down when needed;
    3. Avoiding physical infrastructure like physical data centers, racking, wiring just to name a few;
    4. Going global in minutes compared to traditional model that takes 3-6 months;
    5. Moving from capital expenses to flexible expenses means to facilitate innovation as many times as you want instead of spending money on purchasing hardware
  • AWS follows the "shared-security model" meaning AWS is responsible for the security of the cloud while customers are responsible for the security in the cloud.
  • There are four primary certifications that can be obtained; Developer, SysOps Administrator, DevOps Engineer and Solutions Architect. There are also two different levels of Solutions Architect certification. The first level is Associate, and the highest level is Professional.

    • Brief History of AWS

    "10 years ago, Amazon Web Services was launched with little fanfare as a side business for Amazon.com. Today, it’s a highly successful company in its own right, riding a remarkable $10 billion run rate," according to TechChrunch.com.

    In fact, AWS has grown into the most successful cloud infrastructure company on the planet, garnering more than 30 percent of the market. More than its three closest rivals — Microsoft, IBM and Google — combined, according to data from Synergy Research.


    Speaking recently at an event in Washington, DC, AWS CEO Andy Jassy, who has been there from the beginning, explained how these core systems developed out of need over a three-year period beginning in 2000, and, before they knew it, without any real planning, they had the makings of a business that would become AWS.

    3 Features of Cloud Computing
    1. I.A.A.S. (Infrastructure As A Service) ← Customers have control over all the resources such as VPCs, EC2s, EBS...
    2. P.A.A.S. (Platform As A Service) ← No overhead on customer-side of managing infrastructure like RDS, EMR, Elasticserach
    3. S.A.A.S. (Software As A Service) ← AWS delivers apps over the Net. Such examples are Web-based Email, SalesForce.com...
    3 Types of Cloud Computing
    1. On-Demand ← it works like an electrical switch (turn on and off when needed)
    2. Accessible from the Internet← Resources can be spinned up everywhere
    3. Pay-as-you-go model ← Customers pay exactly for what they use...
    3 Main Services of Cloud Computing
    1. All-in-cloud ← Either customers create or migrate existing apps
    2. Hybrid ← Customers hosts some apps in cloud and on-premises.
    3. On-premises ← Customers deploy their own data centers using virtualization.

    Elasticity

    Amazon VPC Components

    By creating your Virtual Private Cloud standing for VPC which means your own "logically isolated region, you have complete control over the networking."

    • VPCs are region specific meaning for multiple regions, you need to create different VPCs.
    • Subnets are Availability Zone specific meaning for multiple AZs, you need to create multiple subnets.
    • New subnets should be created between CIDR block range of: /19 = 8,190 IP addresses and /28 = 16 IP addresses
    • Route Table (or internal mapping traffic) is automatically set up when creating a VPC. Thus, each subnet should have its own route table.
    • Internet Gateway (IG) allows your VPC to communicate with the Internet
    • NAT Instances are not only located in Public Subnet but also act like a bridge to connect the Private Subnet to the Internet in order to download firmware updates and database patches.
    • NAT Gateway is a managed service which provides better availability and bandwidth and performs the same functions as NAT instances.
    • Egress-Only Internet Gateway allows VPC to communicate with the Internet for IPv6 traffic only.
    • Elastic Network Interface (ENI) is a virtual network interface that you can attach to an instance. ENI's attributes are: MACaddresses, 1 IPv4, 1 or more IPv6, 1 or more security group, a source/destination/check flag and description.
    • Elastic IP address (EIP) is not only a static IP address but also a public IPv4 address.
      Public IP addresses are dynamic whereas Elastic IPs get allocated to your account, and stay the same.
    • Security Group (SG) not only acts as a virtual firewall but also is instance specific meaning different SGs for different instances.
      SGs are stateful means changes applied to incoming rule (allowed Port 80) automatically apply to outgoing rule (opened Port 80).
    • Network Access Control List (NACL) acts as a firewall and is subnet specific.
      NACLs are stateless means changes applied to incoming rule will not be applied to the outgoing rule. e.g. If you allow an incoming port 80, you would also need to apply the rule for outgoing traffic.
    • VPC Peering allows 1 VPC to connect to another using IPv4 or IPv6 addresses. You can't peer VPCs across regions.
    • VPC Endpoint allows you to connect to VPC and other services outside like S3 and Dynamo DB. No public IPv4 and Internet Gateway needed.
    • VPC Flow Logs enables you to capture information about IP traffic going to and from network interfaces in your VPC.
    • Default VPC is created in each region by default. with the following features: Dynamic Public & Private IPs, Public & Private DNS names...
    Elastic Compute Cloud (EC2)
    Networking play an important role in how applications are going to talk to each other. EC2 has the following features:
    • Scalability: you can scale up and scale down at any point.
    • Time to market: EC2 can be deployed almost instantly.
    • Cost effective: you don't really have to pay for a huge capital expense.
    • Reliable: EC2 is 99.95 percent available for each region.

    There are 5 types of EC2 instances available:
    1. General-purpose: suitable for light workloads. Example web servers.
    2. Compute optimized: suitable for high performance computing. Example gaming servers, media transcoding, batch jobs...
    3. Memory optimized: suitable for running in-memory databases.
    4. Storage optimized: suitable for data warehouse applications.
    5. Advanced computing: good for machine learning algorithms, molecular modeling...
    AWS provides multiple pricing options depending on your payment flexibility and instance's duration:
    • On-demand instance: you pay just for the usage with no upfront and hidden costs.
    • Reserved instance: requires either 1 or 3-year commitment with a discount up to 75%.
    • Spot instance: is great for workloads that can restart from where they failed. Discount is up to 90%.
    Storage Options
    AWS provides a wide range of storage options to optimize your applications.
    • Amazon S3 (Standard): suitable for Large objects
      • Amazon S3 Standard Infrequent Access (IA): data is accessed less frequently
      • Amazon S3 Reduced Redundancy Storage (RRS): data is easily reproduced in different video resolutions.
      • Amazon S3 One Zone-Infrequent Access (S3 One Zone-IA): less frequently but requires rapid access when needed.
    • Amazon EC2 Ephemeral Storage: suitable for Transient data
    • Amazon EBS: suitable for Persistent block storage with snapshots
    • Amazon Glacier: suitable for Archive data
    • Amazon CloudFront: suitable for Content distribution
    • Amazon DynamoDB: great for Simple nonrelational data
    • Amazon RDS: Automated, managed MySQL, PostgreSQL, Oracle, MariaDB, SQL Server
    • Amazon Aurora: Cloud-optimized flavors of MySQL and PostgreSQL
    • Amazon Redshift: suitable for Data warehouse workloads
    Databases on AWS
    A database management system (DBMS) is the software that controls the storage, organisation, and retrieval of data. DBMS and RDBMS are often used interchangeably.

    In order for data to be accessible, RDBMS follows certain rules to ensure data integrity:
    • The first integrity states that rows should be distinct.
    • The second integrity states that column values must not have repeating groups or arrays.
    • The third integrity is about the concept of NULL which means a column's value is not known.

    Various ways to host your Database

    • Scenario 1: Hosting the DB in your Data Center (On-Premises) | you have to take care of everything.
    • Scenario 2: Hosting the DB on Amazon EC2 Servers | it is a shared-responsibility.
      • You take care of App optimization, Scaling, HA, DB backups, DB Software patches, OS patches.
      • AWS take care of OS installation, maintenance, racking, power, cooling, networking, and so on...
    • Scenario 3: Hosting the DB using Amazon RDS | AWS does all the heavy lifting for you.

    Hosting Amazon RDS vs Amazon EC2

    You should choose RDS if You should choose EC2 if
    You don't want to manage the databae You need full control over the database
    You lack in-house expertise to manage databases You need operating system access
    Yo want automated backup and recovery You need full control over backups, replication, and clustering
    You want to focus on high-level tuning tasks and optimization You size & performance needs exceeds the Amazon RDS offerings
    You want to push-button multi-AZ replication Your RDBMS engine features are not available in Amazon RDS
    You want to focus on tasks bringing value to your business You want superuser privilege on the database
